# tiledown.com

Public website source for [TileDown](https://github.com/TileDown/tile-down).

The site is generated by the `tiledown` CLI from Markdown content in
[`content/`](content). GitHub Pages publishes the generated artifact to
[`tiledown.com`](https://tiledown.com).

## Repository Layout

- `content/` - source pages, posts, assets, and site configuration.
- `scripts/build.sh` - builds the static site.
- `scripts/check.sh` - builds the site, serves it locally, and runs browser
  checks.
- `.github/workflows/pages.yml` - Linux build and GitHub Pages deployment.

## Build Locally

Use a sibling checkout of `TileDown/tile-down`:

```sh
scripts/build.sh
scripts/check.sh
```

If the engine checkout lives somewhere else, set `TILEDOWN_REPO`:

```sh
TILEDOWN_REPO=/path/to/tile-down scripts/check.sh
```

The generated site is written to `.build/website` by default.

## Deployment

The Pages workflow checks out `TileDown/tile-down`, builds this content on
Linux, runs the browser checks, and deploys the static artifact.
